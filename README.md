# Mail Skill 📧

**Mail Skill** 是一个专为 AI Agent（智能体）打造的本地化、全功能邮件管理技能插件。它赋予了你的 AI 助手处理复杂邮件任务的能力，让 AI 能够像真人秘书一样，安全、高效地为你收取、阅读、总结、归档和发送邮件。

---

## ✨ 项目亮点

- 🛡️ **本地收取与存储，绝对安全**
  - 所有邮件数据（EML 源码、JSON 结构化数据、附件）均拉取并保存在本地 `mail_data` 目录。
  - 使用本地 SQLite 建立索引，检索极速且不再依赖持续的服务器网络请求。
  - 你的邮件数据和账号密码绝不经过任何第三方中间服务器，完全由本地环境掌握。
- 🚀 **全面且强大的邮件功能**
  - **毫秒级极速搜索**：内置本地检索引擎，不需要连接外网或依赖外部服务，就能瞬间在成千上万封邮件中搜出你想要的内容。
  - **对话时间线追踪**：自动把散落在各处的“回复邮件”串联起来。A 回复 B、C 回复 A... 帮你自动还原出完整的聊天时间线，告别碎片化阅读。
  - **优雅的聊天卡片**：将长篇大论的邮件自动整理成整洁的卡片（包含发件人、收件人、摘要、附件等），在 AI 聊天窗口里看着就像微信聊天一样舒服。
  - **自动排版的高颜值发件**：写邮件时直接输出 Markdown 格式，系统会自动将其渲染为带有精美科技风 CSS 的富文本 HTML 邮件（包含漂亮的表格、层级分明的标题和高亮代码块），发送给对方尽显专业。
  - **收发自如**：支持主流的邮件收发，还能直接发送排版精美的图文邮件。
  - **灵活收件**：不仅能收收件箱，还能把服务器上各种分类文件夹里的邮件一并收下来。
  - **智能附件处理**：发送邮件时，你可以直接把一堆文件甚至整个文件夹丢给它，它会自动帮你打包成压缩包发出去。
  - **懂你的回复方式**：支持“回复”和“回复全部”，并自动保持邮件线索不断。
  - **多账号不串味**：你可以同时绑定工作邮箱和私人邮箱，它们的数据和签名完全隔离，互不干扰。
  - **深度兼容**：完美兼容网易系（163/126/yeah）、QQ 等主流邮箱的特殊登录机制。
  - **防重复收取**：智能识别已经收过的邮件，不浪费你的本地空间。
  - **一键整理**：支持标记已读、加星标、移动文件夹、删除等，并在服务器端同步生效。
- 🧠 **AI 专属的持久记忆**
  - **记性好**：告诉它“老板的邮箱是 xxx@xxx.com”，它就会永远记住，下次你只需说“发邮件给老板”。
  - **不忘签名**：你可以为每个账号设置专属的邮件结尾签名。只要设置一次，它以后每次发邮件都会自动帮你加上。
  - **不卡顿**：后台自动默默收邮件，不影响你继续和 AI 聊天。
- 🤖 **专为 AI Agent 设计**
  - **完美兼容**：原生支持并适配市面上各种主流 Agent 工具，如 **OpenClaw、Claude Code、WorkBuddy、OpenCode、Trae** 等。
  - **专业汇报**：内置智能 Markdown 简报生成器，自动识别“验证码”、“重要邮件”、“待回复邮件”，让 AI 每次收信后像真人秘书一样为你进行专业汇报。

## ❓ 为什么要开发这个 Skill？

在日常办公中，我们经常需要处理大量邮件。虽然现代邮箱客户端很多，但它们**缺乏大模型的理解能力**。
当我们试图让 AI 帮我们处理邮件时，往往面临以下痛点：
1. **网络与耗时问题**：AI 每次实时通过 IMAP 搜索服务器非常缓慢，容易触发超时或触发邮箱服务器的频率限制。
2. **上下文过载**：直接把几十封长邮件扔给大模型，会导致严重的 Context 浪费甚至 Token 溢出。
3. **隐私安全**：不想把自己的核心邮箱账号密码交给不可控的云端平台托管。

**Mail Skill** 彻底解决了这些问题：**将繁重的拉取、存储、检索工作交给本地脚本完成，而把最核心的“阅读、提炼、撰写”工作交给 AI Agent**，实现了效率和智能的完美结合。

## 🎯 使用场景

- **📬 每日晨报**：每天早上让 Agent 收取昨晚的所有邮件，并输出一份带验证码、待办事项和重点会议的总结简报。
- **🔍 极速检索**：让 Agent 帮你在过去一年的本地邮件库中，精准找出某个供应商发来的发票附件。
- **✍️ 智能回复**：Agent 读取上下文后，根据你的简短指令（如：“告诉他我下周一有空”），自动拟写并发送回复邮件。
- **🗂️ 批量整理**：让 Agent 帮你把所有包含“退订”或“Newsletter”字样的邮件一键标记已读并移动到“归档”文件夹。

---

## 🔄 核心工作流

AI Agent 使用此技能的标准工作流如下：

1. **获取最新 (Fetch)**：让 Agent 把新邮件拉取到本地。
2. **生成简报 (Summarize)**：收取完成后，Agent 会为你生成一份像早报一样的总结，重点突出验证码、待办事项。
3. **本地检索 (Search)**：问 Agent 某件事，它能瞬间从本地数据库里把相关邮件翻出来。
4. **深入阅读与追踪 (Read/Thread)**：让 Agent 把某封长邮件整理成一张“信息卡片”发给你，或者帮你把前因后果的“聊天记录”全部顺出来。
5. **处理与回复 (Send/Mark/Move)**：动动嘴，让 Agent 帮你回邮件、带上附件，或者把没用的邮件一键归档。

## 📦 安装与配置方式

### 1. 环境要求
- Python 3.8+
- 支持运行 Shell 和 Python 的操作系统（macOS / Linux / Windows）

### 2. 部署步骤
1. **解压项目**：将打包好的 `mail-skill-xxxx.zip` 解压到你的工作目录。
2. **安装依赖**：
   ```bash
   pip install -r requirements.txt
   ```
3. **配置账号**：
   - 复制 `example.env` 并重命名为 `.env`。
   - 填入你的邮箱账号、授权码（或密码）、IMAP 和 SMTP 服务器地址。
   - 支持配置多个账号，只需按照 `MAIL_ACCOUNT_2_`，`MAIL_ACCOUNT_3_` 的前缀继续添加即可。
   ```env
   # --- Account 1 ---
   MAIL_ACCOUNT_1_EMAIL=your_email@example.com
   MAIL_ACCOUNT_1_PASSWORD=你的授权码
   MAIL_ACCOUNT_1_IMAP_SERVER=imap.example.com
   MAIL_ACCOUNT_1_SMTP_SERVER=smtp.example.com
   
   # --- Account 2 ---
   MAIL_ACCOUNT_2_EMAIL=another_email@example.com
   MAIL_ACCOUNT_2_PASSWORD=你的授权码
   MAIL_ACCOUNT_2_IMAP_SERVER=imap.another.com
   MAIL_ACCOUNT_2_SMTP_SERVER=smtp.another.com
   ```
4. **导入 Agent**：
   将本目录或生成的 `.skill` 文件挂载/导入到你的 AI Agent 环境中（如 Trae 的 Skill 面板，或 Claude Code 的工具库中），Agent 会自动读取 `SKILL.md` 掌握使用方法。

## 💡 常见问题 (FAQ)

**Q: 为什么我收取邮件时提示报错或连接超时？**
> A: 请检查你的 `.env` 配置。大多数现代邮箱（如 QQ、网易、Gmail）出于安全考虑，要求使用**应用专用密码（授权码）**而不是登录密码。同时请确认 IMAP/SMTP 服务是否已在邮箱网页端设置中开启。

**Q: 本地存储的邮件在哪里？会不会占用很大空间？**
> A: 邮件默认保存在项目下的 `mail_data` 目录中。包括 `.eml` 原文件、结构化的 `.json` 和附件。如果你定期清理不需要的邮件，占用空间是可控的。你可以随时通过删除 `mail_data` 目录来清空本地缓存，下次收取会重新拉取。

**Q: 一次收取几百封邮件卡住了怎么办？**
> A: Mail Skill 已经将收取过程做成了异步任务。当收取超过 100 封邮件时，系统会要求 `--confirm` 确认。你可以随时使用 `fetch-status <task_id>` 查看后台拉取进度。

**Q: 附件怎么处理的？**
> A: 附件会被自动下载并保存在 `mail_data/attachments/<message_id>` 目录下。Agent 可以通过 `read` 命令获取到附件的具体本地绝对路径，如果你的 Agent 具备读取文件的能力，可以直接解析这些附件（如 PDF、Word）。
