# Research Summary: Mail Skill v2.0

**Researched:** 2026-04-04
**Confidence:** MEDIUM

## Key Findings

### Stack (技术栈)
- **新增依赖**: `dateparser` (自然语言日期解析), `pytest/ruff/mypy` (代码质量)
- **复用现有**: Jinja2 (模板), ChromaDB + CrossEncoder (分类/搜索), http.server (附件服务)
- **不引入**: spaCy, Rasa, FastAPI (过重，不符合项目约束)

### Table Stakes (必备功能)
- 已有: FTS5搜索, 向量搜索, 混合搜索, 邮件线程, 多账户, 附件处理
- 缺失: 附件预览, 富文本详情页, 批量操作

### Differentiators (差异化功能)
- **自然语言搜索** - 核心差异化，高价值
- **智能分类** - 可从规则开始，逐步演进
- **回复模板** - 低复杂度高价值
- **附件预览服务** - 本地HTTP服务器

### Anti-Features (明确不做)
- 云同步, 移动端, 实时推送, 自动化规则, 多语言i18n

## Architecture (架构)

新增 4 个组件:
1. **QueryParser** - NLU查询解析层
2. **EmailClassifier** - 分类引擎
3. **AttachmentServer** - 本地HTTP附件服务
4. **TemplateManager** - 回复模板管理

## Build Order (推荐实施顺序)

1. **代码质量基础** - 测试、类型注解、错误处理
2. **附件预览服务** - 独立功能，低风险
3. **自然语言搜索** - 核心差异化
4. **智能分类** - 规则优先
5. **回复模板** - 利用现有Jinja2

## Critical Pitfalls (关键陷阱)

1. **NLU搜索破坏现有语义** - 需并行实现，保留回退
2. **分类标签漂移** - 固定分类体系，置信度阈值
3. **附件服务安全** - 路径穿越防护，本地绑定
4. **模板未填充占位符** - 发送前验证

## Complexity Estimates

| 功能 | 开发天数 | 风险 |
|------|----------|------|
| 自然语言搜索 | 3-5 | 中 |
| 智能分类(规则) | 2-3 | 低 |
| 回复模板 | 1-2 | 低 |
| 附件预览服务 | 1-2 | 低 |
| 邮件详情优化 | 2-3 | 低 |
| 代码质量加固 | 2-3 | 中 |

## Recommended v2.0 Scope

**优先包含:**
1. 代码质量基础 (测试、类型、错误处理)
2. 自然语言搜索
3. 附件预览服务
4. 智能分类 (规则版)
5. 回复模板
6. 邮件详情优化

**延后到 v2.1:**
- LLM驱动的回复建议
- 智能跟进提醒
- 邮件分析仪表板
