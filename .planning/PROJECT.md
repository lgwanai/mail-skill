# Mail Skill v2.0

## What This Is

Mail Skill 是一个专为 AI 智能体设计的邮件管理增强系统。v2.0 版本将现有功能从"基础工具"升级为"智能助手"，重点增强搜索能力、邮件分类、附件处理和代码质量。

## Core Value

**让邮件管理从"手动操作"变为"意图驱动"** —— 用户只需表达意图（如"上周王总的预算邮件"），系统自动理解并执行。

## Requirements

### Validated

- ✓ IMAP/SMTP 邮件收发 — 现有功能
- ✓ 多账户管理 — 现有功能
- ✓ 本地存储 (SQLite + ChromaDB) — 现有功能
- ✓ 全文搜索 + 向量搜索 + 混合搜索 — 现有功能
- ✓ Markdown 转 HTML 邮件格式化 — 现有功能
- ✓ 邮件线程时间线 — 现有功能
- ✓ 异步邮件收取 — 现有功能

### Active

- [ ] 智能自然语言搜索 — 一句话搜索，自动解析时间、发件人、主题
- [ ] 邮件智能分类 — 自动识别重要程度，分类展示
- [ ] 回复模板系统 — 常用回复快速调用
- [ ] 邮件详情页优化 — 更美观、信息更全面的展示
- [ ] 标记功能增强 — 更灵活的标记和批处理能力
- [ ] 附件预览服务 — 本地 HTTP 服务器，点击链接直接下载/预览
- [ ] 代码质量加固 — 测试覆盖、类型注解、错误处理统一

### Out of Scope

- 腾讯 COS 附件存储 — v2.0 仅支持本地服务器模式，COS 留作后续扩展
- 邮件规则自动化 — 自动分类、自动回复规则暂不纳入
- 移动端支持 — 保持 CLI/AI Agent 使用场景
- 多语言国际化 — 保持中文界面

## Context

### 现有架构
- **客户端层**: `mail_cli.py` (CLI 入口，命令路由)
- **业务层**: `MailClient` (邮件收发), `MailDatabase` (存储检索)
- **存储层**: SQLite + FTS5 + ChromaDB (向量检索)
- **模板层**: Jinja2 (邮件格式化)

### 用户使用场景
深度使用 Agent + mail-skill 办公的用户，日常操作：
1. 收取 → 浏览 → 回复
2. 搜索历史邮件
3. 批量处理（标记、归档、删除）
4. 发送带附件的邮件

### 核心痛点
1. 搜索不够智能，需要多步操作
2. 邮件分类混乱，难以快速识别重要程度
3. 回复效率低，重复内容每次重写
4. 邮件详情展示不够美观全面
5. 附件无法直接预览/下载
6. 标记功能不够灵活

## Constraints

- **技术栈**: Python 3.8+，保持现有依赖最小化
- **兼容性**: 不破坏现有 CLI 命令和 API
- **存储**: 继续使用本地存储，不引入云服务依赖
- **性能**: 智能搜索响应时间 < 2秒
